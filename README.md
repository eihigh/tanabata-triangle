# 七夕トライアングル (Tanabata Triangle)

3人用の隠れんぼ系ボードゲームのブラウザ実装（ローカル1画面・パス回し）。

- **織姫・彦星（シーカー）**: 出会えれば勝ち。互いの盤面は見えず、軌跡の交差ヒントを頼りに近づく。
- **王様**: 二人を出会わせなければ勝ち。両盤面を見て、各シーカーの移動前にデブリを置いて妨害する。

> **既定（確定版）の対戦設定**: 盤面 **7×7**・移動は **織姫1d4 / 彦星1d6**・出目は **王様のみ公開(king)**・
> 初期中央デブリ **1個**・王様のデブリは常に **1手番1個**。全AI最善どうしでシーカーのクリア率は約 **65%**
> （王様にも十分勝機がある狙いのバランス）。開始画面の初期選択もこの設定になっている。

## ルール

- 共有座標グリッド（9×9）上に織姫・彦星の2駒がいる。各シーカーは**自分の盤面だけ**を見る。王様は両方を見る。
- シーカーは1手で**上下左右にちょうど3マス**移動する（途中停止不可、自マスの重複・折り返し可、盤外・デブリへは進入不可）。
- 通ったマスには**軌跡**が残る。`織姫の軌跡 ∩ 彦星の軌跡` が**交差ヒント（★）**として各手番の開始時に開示される。
- 王様は各シーカーの移動前に、その盤面の**軌跡が無いマス**にデブリを1個置ける（2盤面で別々の位置に置ける）。
  ただし**相手シーカーの現在位置には置けない**（合流点を毎ターン封鎖してシーカーを必勝阻止する致命的手の禁止）。
- 手番の流れ: `織姫デブリ → 織姫移動 → 彦星デブリ → 彦星移動 →（ヒント更新）→ …` を **7ラウンド**。
- あるシーカーが移動を終えた時点で**両者が同じマスに止まっていればシーカーの勝ち**。会えなければ王様の勝ち。

パス回しのため、各手番の切り替え時にハンドオフ画面を挟んで隠し情報の覗き見を防ぐ。

## AI と役の選択

開始前の画面で **織姫・彦星・王様それぞれを「人間 / AI」から任意に組み合わせて**選べる（全AIの観戦も可）。
AI は確率分布で手を選ぶ:

- **シーカーAI**（織姫/彦星）: 相手の盤面は一切覗かず、**相手の現在位置のベイズ的信念分布**を作って手を選ぶ。
  使う情報は正当なものだけ:
  - 到達可能性＋パリティの硬い制約（相手の開始位置・移動量・完了手数は公開設定なので、
    「k手×移動量以内かつ偶奇の合うマス」以外は確率0＝盤の約半分を即除外）
  - **負の情報**: 相手が自分の軌跡上にいればヒントに必ず現れる → 「自分の軌跡のうちヒントに無いマス」に
    相手はいない（確率0）。この“交差していない＝通っていない”情報の活用が挙動改善の核。
  - 正の情報: ヒント周辺の確率の山、共有焦点（ヒント重心）への前進事前分布。
  移動評価は「着地＝即合流の確率」「相手への期待距離」「共有焦点への収束」＋**新規マス探索ボーナス**
  （自軌跡上をぐるぐるせず情報を取りに行く）をソフトマックス分布でサンプリング。
- **王様AI**: 両盤面を見られる立場をフル活用し、**実際の両者の位置**に基づいて
  「この手番で着地合流される脅威の遮断」「相手位置の隣接（着地急所）封鎖」「二人の間の回廊封鎖」
  「接近妨害」を評価してデブリ先を選ぶ（同じくソフトマックス分布）。
  さらに **合流点の先読み**を行う: 両者の位置と期待歩数から「二人が最終的に出会いそうな地点」を
  速度で内分して予測し（速い駒ほど遠くまで詰めるので遅い側寄り）、**序盤（最初の1〜2手番）だけ**
  その予測地点＝多くは中央付近を先取りして塞ぐ。序盤は駒が遠く「その手番の着地遮断」がまだ効かないため、
  空いた序盤に合流予測点を固めるのが効く。なお“中央にただ固めるだけ”の単純戦法は弱く（終盤の実着地を
  止められない）、**先読みは既存の戦術的遮断に上乗せして初めて強い**（この上乗せが王様AIを実力反映のまま底上げする）。

**人間シーカーがいる対戦では、AIの手番中は盤面を伏せる**（人間シーカーへの情報漏れ防止）。
人間が王様のみ、または全員AIの場合は、観戦のためAIの手番も盤面を表示する。

## 遊び方

ビルド不要。静的配信して `index.html` を開くだけ。

```sh
python3 -m http.server 8000
# → http://localhost:8000/index.html
```

（`file://` 直開きは ES モジュールの CORS 制約で動かないため、簡易サーバー経由で開くこと。）

## 構成

- `index.html` / `css/style.css` — 画面
- `js/engine.js` — ルールエンジン（純粋・DOM非依存の ES モジュール）
- `js/ai.js` — 確率分布ベースの AI（純粋・DOM非依存）
- `js/render.js` — Canvas 描画
- `js/app.js` — 役選択・フェーズ進行・入力・ハンドオフ・AI進行・勝敗
- `tests/engine.test.mjs` / `tests/ai.test.mjs` — 単体テスト
- `SIMULATION.md` — バランス調整の探索・シミュレーション記録（確定設定に至る根拠、王様AIの合流点先読みなど）

### 調整可能な定数

`js/engine.js` の `DEFAULTS` で変更できる。**既定値は開始画面の確定設定に一致**しており、
`createGame()` を引数なしで呼ぶと **7×7 / 織姫1d4・彦星1d6 / 出目king / 初期中央デブリ1 / 7ラウンド** になる。

- `BOARD_SIZE`（既定 7）/ `STEPS: { orihime, hikoboshi }`（既定 `'d4'`/`'d6'`。数値 or `'d4'`/`'d6'`）/ `MAX_ROUNDS`
- `PUBLIC_ROLLS`: 出目の公開範囲。既定 `'king'`。`'all'`（=`true`）/ `'king'` / `'none'`（=`false`）。
- `INITIAL_CENTER_DEBRIS`: 初期中央デブリ。既定 `true`（1個）。数値→その個数を中心に固めて配置（両盤同一＝シーカー共通知識）。
- `FIRST_ROUND_DEBRIS`: 初日（ラウンド1）のみ王様が置ける個数（未指定なら `DEBRIS_PER_TURN` と同じ）。
- `STEPS` を丸ごと省略／`{}` 指定にすると両者 `STEPS_PER_MOVE`（既定3, 固定）にフォールバックする。

> 開始画面（`js/app.js` の `variant`）も同じ既定を初期選択にしている。シミュレーター（`tools/simulate.mjs`）は
> 歴史的な比較のため `public` を明示指定（既定 `all`）し、`combos` も各バリアントの盤サイズ・移動量を明示している。

## シミュレーター（シーカー勝率）

全AI対戦をモンテカルロで多数回まわし、**乱数混入率 `epsilon`**（0=AI本来の分布〜1=完全ランダム）ごとの
シーカーのクリア率と、**日毎（ラウンド毎）の累積クリア率**を出す。隠れ情報＋2盤面の軌跡/デブリ集合という
状態空間の巨大さから厳密DPは非現実的なため、**各設定に壁時計タイムアウトを設けたモンテカルロ**にして、
常に有限時間で終了し完了試行に対する率を報告する。

```sh
node tools/simulate.mjs                               # 既定（epsilon 0〜1 を掃引）
node tools/simulate.mjs games=4000 budgetMs=6000      # 試行数/タイムアウト
node tools/simulate.mjs target=seekers                # 乱数混入をシーカーだけに（king/all も可）
node tools/simulate.mjs board=7 ori=d4 hik=d6         # 盤サイズ・移動量（数値 or 'd4'/'d6'）
node tools/simulate.mjs public=king                   # 出目公開 all|king|none
node tools/simulate.mjs centerDebris=2                # 初期中央デブリ個数
node tools/simulate.mjs firstRoundDebris=2            # 初日のみ王様が置ける個数
node tools/simulate.mjs combos=1                      # バリアント一括比較
node tools/simulate.mjs matrix=1                      # シーカーeps × 王様eps の 3×3
```

結果は表形式で標準出力し、`tools/sim-result.csv` にも書き出す。
（開始位置が対角コーナー＝マンハッタン距離16のため、最短でも3ラウンド目まで出会えず Day1/Day2 は必ず0%。）

**バランス調整の探索・計測結果（確定設定に至る根拠、各バリアントのクリア率、王様AIの合流点先読みなど）は
[SIMULATION.md](SIMULATION.md) にまとめている。**
