# 七夕トライアングル

七夕をモチーフにした2人協力の位置推理ボードゲーム。織姫と彦星が、互いの位置が見えない 7×7 グリッド上で相手を探し、7日以内に**同じマス**に立てたら再会（勝利）。

手がかりは2つ:

- **軌跡の交差** — 自分が移動した経路が相手の**直前の手番**の経路と重なると、その交差マスだけが両者に開示される（✦）
- **出目の相互開示**（オプション） — 手番終了時に自分の出目（合計値）を相手に伝える

移動はサイコロ2個の合計**ちょうど**の歩数（桃鉄式）。上下左右に1マスずつ、途中では止まれない。

## ファイル

| ファイル | 内容 |
|---|---|
| `tanabata-triangle.html` | デジタルプロトタイプ（パス＆プレイ、1端末を回して遊ぶ）。ブラウザで開くだけ |
| `sim.js` | バランス調整用シミュレータ（Node.js、依存なし） |
| `SIMULATION.md` | シミュレーション実験の結果と設計への示唆 |
| `DESIGN.md` | 仕様・設計判断の履歴・引き継ぎ資料 |

## 遊び方

`tanabata-triangle.html` をブラウザで開く。自分の番だけ盤面が見える（相手の位置・軌跡は非表示）。手番が終わったら端末を相手に渡す。

## シミュレータ

```sh
# 単発: <試行数> <ダイス> <盤> <日数> <減衰>
node sim.js 10000 2d6 7 7 1 --policy=greedy --share

# 実験マトリクス一括（方策×出目開示、盤・日数感度、相手モデル、ε感度）
node sim.js --matrix 10000
```

方策は `random`（無情報）/ `greedy`（相手位置のベイズ推定へ最短接近）/ `infogain`（交差を狙って情報収集）/ `hybrid`（序盤infogain→終盤greedy）。`--eps=0.4` で「確率0.4で無情報に動く」人間らしい不完全さを混ぜられる。詳細は `sim.js` 冒頭コメントと `SIMULATION.md` を参照。
