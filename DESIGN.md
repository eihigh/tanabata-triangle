## 1. ゲームのコンセプト

七夕をモチーフにした、位置推理ゲーム。織姫と彦星が、互いの位置が見えないグリッド上で相手を探し、出会えれば勝ち。海戦ゲーム（バトルシップ）の「隠れた相手を手がかりから絞り込む」推理と、桃太郎電鉄の「サイコロの出目ちょうどで目的地に止まる」位置合わせを掛け合わせた点が核。

当初は3人ゲーム（織姫・彦星・おじゃま係）として構想されたが、まず2人協力の核が面白いかを固める方針で、現行プロトタイプは**2人協力版**になっている。おじゃま係は将来的に戻して3人対戦版に拡張する（第6節）。

### 手がかりの仕組み（このゲームの心臓部）

プレイヤーは自分の位置と自分の軌跡しか見えない。相手の位置は不明。唯一の手がかりが「軌跡の交差」。**自分が動いた手番で、自分の経路（通過した全マス）が相手の直前の軌跡と重なったとき、その交差したマスだけが、横切った本人に開示される**。被さられた側には伝わらない（＝知れるのは「今、相手の直前の道を横切った側」だけ。自分の古い軌跡を相手が踏んだことを次の自ターンで知る、といった逆方向の情報は起きない）。これを積み重ねて相手の居場所を推理する。

## 2. 確定した最終仕様（現行プロトタイプ）

現行のデジタルプロトタイプ（`tanabata-triangle.html`）に実装済みの確定パラメータ:

| 項目 | 値 | 補足 |
|---|---|---|
| グリッド | 7×7 | 外周24マスがスタート候補 |
| プレイヤー | 2人（織姫・彦星）協力 | おじゃま係は現状なし |
| スタート位置 | 各自、外周のランダムなマス | 相互不可視。初期は最小距離 `max(4, round(N*1.1))` = 8 マス離す |
| 移動 | サイコロ2個の合計（2〜12）ちょうど | 途中では止まれない（桃鉄式）。合計値ぴったりで着地 |
| 移動の性質 | 上下左右4方向、1マスずつ。自己交差可、盤外不可 | 「ちょうどk歩」で到達できるマスにのみ着地可能 |
| 交差判定 | **自分の移動 × 相手の直前の移動（直近1手番）** の重なりのみ。開示は横切った本人だけ | 減衰=1。過去の全軌跡ではない（後述の理由）。相手には伝わらない |
| 勝利条件 | 二人が**ぴったり同じマス**に立つ | 隣接では不可（同マス限定） |
| 敗北条件 | 7日（=7ラウンド）以内に会えなければ「また来年」 | 時間切れ。2人協力なので実質「時の流れが敵」 |
| 手番順 | 織姫 → 彦星 で1日、これを7日繰り返す | パス＆プレイ（1端末を回す） |

### コード上の定数（tanabata-triangle.html）

```
const N = 7;        // グリッド一辺
const MAX_DAY = 7;  // 日数
const NDICE = 2;    // サイコロ本数
const FACES = 6;    // 面数
const DECAY = 1;    // 交差判定：直近何手番の相手軌跡を有効とするか（1=直前のみ）
```

### 状態管理の要点（実装リファレンス）

- `positions`: {orihime:[r,c], hikoboshi:[r,c]}
- `trails`: 各プレイヤーごとに `Map<"r,c", turn>`。マスに「最後に通った手番の通し番号」を刻む。減衰判定に使う。
- `turn`: 手番の通し番号（織姫・彦星の各手番でインクリメント）
- `knownCross`: 各プレイヤーが**自分の手番で横切って見つけた**交差マスの `Set<"r,c">`（表示用に永続保持。相手の見つけた交差は入らない）
- 交差判定: 移動した経路 `segment` の各マスについて、相手trailに記録があり、かつ `turn - t <= DECAY` なら交差成立。成立した交差は**動いた本人の `knownCross` にだけ**追加し、相手への申し送り（`news`）には含めない（出目のみ伝える）
- 到達マス計算 `reachableSet(start,k)`: kステップBFSで「ちょうどk歩で行けるマス集合」をDPで求める（経路列挙はしない=高速）
- 経路復元 `onePath(start,target,k)`: 選んだ着地マスへの経路を1本だけBFSで復元（交差判定に通過マスが必要なため）

## 3. 検討中の次の変更：出目の相互開示

【実装済み。効果の定量化は SIMULATION.md 実験1・5 を参照】

案: 盤面（相手の位置）は見えないままだが、**相手が振った出目（合計値）は相手に伝えてよい**ことにする。

### 期待される効果

- 相手の出目が分かると、交差ヒントの価値が跳ね上がる。「前の手番であの地点で交差した＝相手はその辺にいた」＋「相手は今9を出した」から、その交差点を中心に**マンハッタン距離9・パリティ（偶奇）の合うマス群**へ現在地を絞れる。
- 「ちょうど止まる」制約と噛み合う。相手が9を出しても、盤端や折り返しで9ちょうど止まれる場所は限られるので、候補がさらに絞れる。
- 情報戦が海戦ゲーム的に深まる。

### 設計上の注意

- これは**難度を easy 側に倒す**（ヒント増）。簡単になりすぎたら、盤拡大・日数短縮・おじゃま復活で締める。
- パス＆プレイでは「手番終了時に自分の出目を相手に伝える」表示を足す。アナログなら口頭で「9」と言うだけで運用でき、むしろアナログ親和性が高い。
- **重要**: 現行のランダム行動シミュでは出目共有の効果は測れない（ランダムは情報を使わないため、出目あり/なしで出会い率が変わらない）。効果の定量化には第4節の「賢いAI」が必須。

## 4. 賢いAIの実装方針

【実装済み（sim.js）。以下は設計方針の原文】

### 4.1 各プレイヤーが保持すべき信念（belief）

相手の現在位置についての**確率分布**（または候補集合）を、盤上49マスに対して持つ。

- 初期: 相手は外周24マスのいずれか（自分の初期最小距離制約を加味して事前分布を作ってもよい）。
- 毎手番、以下の観測でこの分布を更新（ベイズ更新／候補集合の絞り込み）。

### 4.2 観測と更新

観測できる情報は次の通り。これらで belief を更新する:

1. **交差の有無と位置**（現行ルール）。自分が移動した経路上で、相手の直前軌跡と重なったマスが分かる。
   - 交差した → 相手は「直前の手番でそのマスを通った」。相手の**1手番前の位置**がそのマス近辺に強く制約される。そこから相手が次にどう動いたかを、相手の移動モデル（4.4）で前方伝播する。
   - 交差しなかった → 自分の通った経路マスは、相手の直前軌跡ではなかった。負の情報として候補から除外（弱いが効く）。
2. **相手の出目**（第3節を採用する場合）。相手が直前に合計 s を出した。
   - 相手の「1手番前の位置候補」それぞれから、ちょうど s 歩で到達できるマスへ確率を配る。パリティと盤端制約で候補が絞れる。
3. **自分が出会っていないという事実**。自分の現在位置に相手はいない（同マスなら勝敗判定で終了しているはず）。自分の現在マスを候補から除外。

### 4.3 行動選択（方策）

belief を踏まえて、自分の着地マスを選ぶ。候補になる目的関数:

- **期待距離最小化**: 相手位置の確率分布の重心（または最尤マス）へ、今の出目でちょうど到達できるマスのうち、次手番以降で出会う期待手数が最小になるマスを選ぶ。
- **情報利得最大化（探索寄り）**: 交差を起こしやすい経路（相手候補が濃い領域を横断する経路）を選び、次手番の belief を鋭くする。
- **ハイブリッド**: 序盤は情報利得重視（相手を絞る）、終盤（残り日数が少ない）は期待距離最小化（会いにいく）。残り日数で重みを切り替える。

協力ゲームなので**両プレイヤーが同じ戦略・同じbelief更新規則を共有**でき、暗黙の協調が可能（相手も自分と同じ推論をしていると仮定して動ける）。これは強力なので、AIはこの前提を使う。ただし通信は禁止（交差と出目以外の情報交換なし）。

### 4.4 相手の移動モデル

belief 更新には「相手がどのマスに動くか」の予測モデルが要る。自己参照的（相手も同じ賢いAI）にすると理想だが計算が重い。段階的に:

- **v1**: 相手はランダムに動くと仮定して belief 更新（実装が軽い。まず動かす）。
- **v2**: 相手も期待距離最小化で動くと仮定（自己整合。重いが実像に近い）。
- v1→v2で出会い率がどう変わるかも興味深い比較になる。

### 4.5 実装構造

`sim.js` に実装済み:

- `makeBelief(board, myStart, minDist)` → 初期分布
- `observeBelief(...)` 交差・負の情報・出目による更新
- `propagateBelief(...)` 相手移動の前方伝播（出目開示あり=正確な遷移行列、なし=2d6混合）
- `chooseMove(...)` → 着地マスと経路
- 方策 `--policy`: random / greedy / infogain / hybrid、`--share` で出目開示、`--opp` で相手モデル v1/v2、`--eps` で不完全プレイ
- `--matrix` で実験マトリクス一括実行

## 5. シミュレーション計画

【実行済み。結果は SIMULATION.md】

すべて 7×7・2d6・7日・減衰1・同マス限定・2人協力 を基準に、以下を変数として振る。各条件 10,000 試行程度で安定。

### 測る指標

- 出会い率（%）
- 平均決着日（会えたゲームのみ）
- 交差/game、交差の起きたゲーム率
- 詰み手番/game（ちょうど止まれず動けない回数）

### 実験マトリクス

1. **方策 × 出目開示**: {random, greedy, infogain, hybrid} × {出目なし, 出目あり}。これがメインの表。出目開示の価値と、賢さの価値を切り分ける。
2. **盤サイズ感度**: 上位方策で N ∈ {5,7,9}。難度が動いたときの調整代を見る。
3. **日数感度**: N=7 で MAX_DAY ∈ {5,6,7,8}。目標出会い率に合う日数を決める。
4. **相手モデル v1/v2**: greedy 方策で相手モデルを切り替え、出会い率の差を見る。

### 目標バランス（仮）

2人協力の賢いプレイで出会い率 **55〜70%** あたりが「難しいが理不尽でない」帯と仮定。ここを外れたら日数・盤・出目開示で調整。（→ 結論: 最適プレイは99%に飽和するため、この帯は「人間らしい不完全プレイ（ε≒0.5〜0.6）」の帯として自然に実現される。SIMULATION.md 参照。）

## 6. 3人対戦版への拡張（おじゃま係）

2人協力の核が固まったら、おじゃま係を戻して当初構想の3人対戦にする。

### これまでの検討で決まっていたおじゃま仕様

- おじゃま係は**全知**（二人の位置・軌跡・交差、二人の距離まで見える）。全知にした理由: 手探りだと手番が作業になり能動性が出ないため。非対称情報を逆手に取り、追う側/追われる側の緊張を作る。
- 妨害手段: 自分の手番に**ブロックを1個設置**（通行・停止不可のマス）。上限は暫定5個。
- ジレンマ: ブロックを二人の中間に置くと足止めになる一方、位置の**逆ヒント**にもなりうる。
- 勝利条件: 7日（or 4日）以内に二人を会わせなければおじゃまの勝ち。

### シミュで測るべきこと

- 全知おじゃまが最短路を塞ぐ戦略を実装し、2d6・7×7・7日で出会い率がどれだけ下がるか。
- おじゃま強度（ブロック上限3/5/7、ブロックが数日で消える減衰の有無）で難度を調整。
- 日数×おじゃま強度の組み合わせ最適化（例: 短い日数ならおじゃま弱め）。
- ※ 実装時の注意: 障害物が入ると「ちょうどk歩到達＝距離＋偶奇」の高速判定が使えなくなるため、`sim.js` の到達判定を層状DPに差し替えること（コード内コメント参照）。

### 敗北条件についての決定事項

無限にターンを続ければ必ず出会えるため、勝利条件ではなく**敗北条件（時間切れ）**が必須という結論に至っている。現行は「7日で時間切れ」。七夕モチーフ（一年に一度）とも噛み合う。

## 7. 設計判断の履歴（なぜこの仕様に至ったか）

将来「なぜこうしたか」を再検討できるよう、経緯と根拠データを残す。すべてランダム行動・おじゃま抜き・同マス限定（特記なき限り）のベースライン測定値。

### 7.1 盤サイズとダイスの変遷

- **初期案 7×7・1d3・7日**: 出会いやすく、かつ1〜3歩の軌跡が短すぎて交差がほとんど起きなかった（交差の起きたゲーム率 34%、交差0.73/game）。「軌跡ゲーム」として成立せず。
- **教訓**: 交差を増やすには「長い軌跡」が要る。軌跡が長いほど相手軌跡と重なりやすい。→ ダイスを大きく。
- **7×7・1d6・7日**: 出会い率61.6%、交差3.96/game・75%。交差は濃くなったが狭くて出会いやすい。
- **盤を広げると**（1d6維持）出会い率も交差も下がる（すれ違わなくなる）。9×9で交差2.5、11×11で交差1.35。広げるだけでは軌跡が絡まない。
- **本命の発見 — 2個ダイス**: 「5×5でダイス2個（2〜12、中央7に集中）」の案。狭い盤を毎回大きく動くので軌跡が絡み、かつ大きく動くほど『ちょうど止まる』位置合わせが効く。探索と位置合わせの両立。

### 7.2 クリア条件：隣接 → 同マス限定

- 隣接可（マンハッタン≤1）だと的が広く運要素が強い。同マス限定（=0）にすると的が1マスに絞られ、桃鉄式の位置合わせが本当に効く。
- 4日・7×7・1d6で、隣接可 40% → 同マス限定 10.9% と出会い率が急落。同マスのほうが「推理して正確に合わせる」技術ゲームになる。→ **同マス限定を採用**。

### 7.3 日数：7 → 4 → 7

- 7日は緩め（同マス・1d6で出会い21%、交差7.4）、4日はハード（出会い11%、交差2.4）。一度4日にしたが、交差が減りすぎ運が絡む。最終的に2d6で交差が潤沢になったため7日に戻して落ち着いた。

### 7.4 交差の減衰：全軌跡 → 直近1手番

- 全軌跡有効だと後半は交差だらけ（7日で交差7.4）。しかも古い交差は「相手が昔通った跡」で現在地と無関係なノイズ。
- 減衰を入れて比較（7日・1d6）: 全軌跡7.5 / 直近3手番4.1 / 直近2手番2.3 / 直近1手番2.3。直近型ほどヒントが「相手の今の居場所」に直結してクリーン。
- **決定打（アナログ化）**: 減衰1手番以外はアナログゲームにしづらい。全軌跡や直近2以上は過去の道を記録し続ける必要があり卓上で無理。減衰1なら「自分が今動いた道すじだけ相手に見せて重なりに印」で運用できる。→ **減衰1手番を採用**。デジタルとアナログで挙動が一致するのも利点。

### 7.5 最終確認データ（採用構成）

- **5×5・2d6・7日・減衰1**: 出会い率44.0%、交差12.5/game、交差起きた率99.9%、詰み0.00。
- **7×7・2d6・7日・減衰1**: 出会い率24.9%、交差8.66/game、交差起きた率98.7%、詰み0.00。
- 7×7を採用。理由: 2d6の期待値7が盤の一辺7と噛み合い、軌跡がうねりすぎず端から端まで動ける。的が49マスで歯応えが出て、探索感が強い。5×5はやや易しく短時間向け（初級プリセット候補）。
- 両構成とも詰み手番0.00。「狭い盤で大きく動くと止まれないのでは」という懸念は、2個ダイスの合計値のばらつきで到達マスが散るため杞憂だった。
