# シミュレーション結果（賢いAI編）

引き継ぎ書（DESIGN.md）第4〜5節の計画に基づき、belief（相手位置の確率分布）を持つ賢いAIを実装して実験マトリクスを回した結果。基準構成は **7×7・2d6・7日・減衰1・同マス限定・2人協力**、各条件 10,000 試行（`node sim.js --matrix 10000 --seed=42`）。

## 交差の開示ルール（重要・確定仕様）

交差は **「今、自分が動いて相手の直前の道を横切った側」だけに開示される**。被さられた側には伝わらない。つまり:

- ✅ 採用: 自分の手番で移動し、自分の経路が相手の直前の軌跡と重なったら、その交差マスを**自分だけ**が知る
- ❌ 除外: 自分の古い軌跡に相手が後から被さり、それを自分の次の手番で知る（逆方向の情報）

この情報構造をシミュにも反映してある（交差は動いた側の belief だけを更新し、相手の belief には交差情報を与えない）。相手が得るのは「相手が1手番動いた」事実と、出目開示ありならその出目だけ。

## ベースライン再現の確認

まずランダム方策で引き継ぎ書 7.5 節の実測値と照合した:

| 構成 | 今回 | 引き継ぎ書 |
|---|---|---|
| 7×7 出会い率 | 24.6% | 24.9% |
| 5×5 出会い率 | 43.3% | 44.0% |
| 7×7 交差/game | 9.4 | 8.66 |
| 詰み/game | 0.00 | 0.00 |

出会い率はよく一致（ランダム方策は情報を使わないので開示ルールの影響を受けない）。**再現OKとみなす。**

## AIの構成

- **belief**: 相手位置の49マス確率分布。初期値は「外周かつ自分から最小距離以上」で一様
- **動いた側の更新**: ①交差あり→相手経路がそのマスを通った（距離減衰の尤度）、②交差なし→踏んだマスに相手はいない（負の情報・確定除外）、③出会っていない→着地マス除外、④出目開示ありなら相手の直前の出目で交差点からの距離を制約
- **動かなかった側の更新**: 交差は伝わらないので、相手（動いた側）の移動を遷移行列で前方伝播（出目開示ありは正確、なしは2d6分布で混合）＋「未出会い」除外のみ
- **方策**: greedy（期待距離最小化＋着地マスの的中確率ボーナス）/ infogain（beliefの濃い領域を横断する経路を選ぶ）/ hybrid（1〜3日目infogain→4日目以降greedy）

## 実験1: 方策 × 出目開示（メイン）

| 方策 | 出目なし | 出目あり | 平均決着日 |
|---|---|---|---|
| random | 23.8% | （同一） | 4.03 |
| greedy | **93.9%** | **95.4%** | 3.72 / 3.38 |
| infogain | 46.0% | 46.4% | 4.06 |
| hybrid | 89.8% | 89.8% | 4.40 / 4.33 |

**発見①: 賢さの価値は依然として大きい。** random 24% → greedy 94%。片側だけが交差を知る非対称ルールでも、動くたびに相手の直前の道の手がかりを積めば、7日でほぼ会える。推理が勝率を作る実力ゲームである点は変わらない。

**発見②: 出目開示に、今度は明確な価値が出た。** greedyで +1.5pt、平均決着が 3.72日 → 3.38日 と 0.34日ぶんも早まる。前バージョン（両者開示）では出目開示はほぼ中立（+0.2pt）だったが、開示を片側だけに絞ったことで情報が希少になり、「交差点＋相手の出目＝距離の輪」という追加の絞り込みが効くようになった。**出目開示を採用する意味がデータで裏づけられた。**

**発見③: infogain単独は崩壊（46%）。** 前バージョンでは交差情報が両者共有だったので情報収集の価値が相対的に低く97%を保っていたが、片側開示では「情報を取りに行っても相手には渡らない・自分が会いに行かないと詰む」ため、序盤を探索に費やすと終盤の接近が間に合わなくなる。**このルールでは『会いに行く』greedyが正解**で、hybridも純greedyに劣る。探索フェーズは不要どころか有害。

## 実験2: 盤サイズ感度（hybrid+出目、7日）

| 盤 | 出会い率 | 平均決着日 |
|---|---|---|
| 5×5 | 92.9% | 3.60 |
| 7×7 | 89.8% | 4.33 |
| 9×9 | 87.3% | 4.70 |

盤を広げると素直に難化する（前バージョンは推理が強すぎて盤を広げても吸収していたが、片側開示では情報が減った分ちゃんと効く）。※hybridは実験1の通り最適ではないので、絶対値より傾向として読む。

## 実験3: 日数感度（7×7、hybrid+出目）

| 日数 | 出会い率 |
|---|---|
| 5日 | 80.1% |
| 6日 | 83.1% |
| 7日 | 89.8% |
| 8日 | 91.2% |

日数が効くようになった（5日80%↔8日91%）。日数は難度調整の実効レバーとして使える。

## 実験4: 相手モデル v1（ランダム仮定）vs v2（greedy仮定）

| 条件 | v1 | v2 |
|---|---|---|
| greedy | 93.9% | 94.9% |
| greedy+出目 | 95.4% | 96.0% |

今度は v2（相手も自分に寄ってくると仮定）が僅かに勝る（+0.6〜1.0pt）。片側開示で自分の belief が薄くなった分、「協力相手も同じくこちらへ寄る」という前提が伝播の精度を上げる。とはいえ差は小さく、実装コストを考えれば **v1でも実用上十分**。

## 実験5: ε感度＝人間らしい不完全プレイ（7×7、greedy、7日）

確率εで無情報（ランダム）に動くAI。人間の「毎回は完璧に推理できない」をモデル化。

| ε | 出目なし | 出目あり |
|---|---|---|
| 0.0 | 93.9% | 95.4% |
| 0.2 | 83.3% | 84.6% |
| 0.4 | **67.8%** | **69.5%** |
| 0.6 | 47.9% | 50.0% |
| 0.8 | 32.6% | 32.8% |

**目標帯55〜70%は ε≒0.4〜0.5（「半分弱は勘、半分強は推理を活かす」プレイ）に相当。** 前バージョンより緩いεで目標帯に入る＝片側開示のほうがゲームが締まっている。出目開示は各εで一貫して +1〜2pt 効く（人間には「距離の輪」の足場を与える価値がさらに大きいはずで、AIの数字は下限とみてよい）。初見はε0.6〜0.8帯（33〜50%）から始まり、上達で95%まで伸びる健全なスキルカーブ。

## 実験6: 収束点コンベンション（focal）の脅威 ★重要

「二人とも盤の中心へ向かってホバリングし、居合わせたら勝ち」という事前の約束事だけの戦略（交差も出目も belief も一切使わない）。協力・隠れ情報ゲームが共通して抱える **Schelling point（焦点）による “必勝コンベンション”** の検証。

| 条件 | 出会い率 | 平均決着日 |
|---|---|---|
| focal 7×7 7日 出目**なし** | 98.2% | 2.49 |
| focal 7×7 7日 出目**あり** | 98.2% | 2.49 |
| focal 9×9 4日（盤拡大＋日数短縮） | 87.7% | 2.32 |
| （対照）greedy+出目 7×7 7日＝正規の推理プレイ | 95.4% | 3.38 |

**発見①: 収束点戦略は正規の推理プレイより強い。** focal 98.2% > greedy 95.4%、しかも決着は 2.5日 と圧倒的に速い。「中心へ行くだけ」という思考停止コンベンションが、交差を積んで推理するプレイを上回る。**推理ゲームとして致命的な支配戦略**。

**発見②: 犯人は出目開示ではない。** focal は出目開示の有無で完全に同一（98.2%／98.2%）。focal は相手の出目を使わないので当然。**「出目開示ありだから必勝法が生まれる」は誤診**——出目開示を消しても収束点必勝は 1pt も動かない。出目開示はあくまで正規の推理プレイ（greedy）を +1.5pt 助ける道具で、必勝コンベンションとは無関係。

**発見③: 盤・日数では潰せない。** 9×9・4日まで締めても focal は 87.7%。盤や日数で締めると、収束点必勝を潰す前に正規の推理プレイのほうが先に成立しなくなる（推理の余地ごと消える）。**盤・日数はこの問題の有効レバーではない。**

**発見④: 唯一の実効的な対策は、盤の対称性を壊す＝おじゃま係。** 収束点（中心）が存在する限りコンベンションは成立する。これを崩すには、中心を占拠・封鎖して予測可能な収束を罰する**全知のおじゃま係（第6節）**が直接の解。おじゃまが全知に設計されている理由がここで裏づけられた——収束してくる二人を追い、焦点を潰せる唯一の存在。

→ **結論: ユーザーの直感は半分正しい。「おじゃまあり必須」は妥当（収束点コンベンションを潰す唯一のレバー）。ただし「出目開示なし必須」は誤り（出目開示は必勝法の原因ではなく、消すと正規の推理が弱くなるだけ）。** 2人協力版は「中心へ行くだけ」という解が存在する砂場であり、推理を本当に意味あるものにするのは3人おじゃま版。おじゃま係は拡張オプションではなく核心である、という位置づけが強まった。

## 設計への示唆（まとめ）

1. **現行仕様（7×7・2d6・7日・減衰1・同マス・片側開示）は良好。** 最適プレイ94%・目標帯はε0.4〜0.5で自然に達成。片側開示により、盤サイズ・日数がちゃんと難度調整のレバーとして効くようになった（実験2・3）。
2. **出目開示は採用推奨**（プロトタイプ実装済み・トグルあり）。片側開示ルールと組み合わさって初めて明確な価値（決着0.34日短縮、各εで+1〜2pt）が出た。アナログでも口頭で言うだけ。
3. **方策は素直な greedy（会いに行く）が正解。** infogain/hybridの探索フェーズはこのルールでは逆効果（実験1）。プレイヤー向けの指針も「手がかりが出たら迷わず寄れ」でよい。
4. さらに締めたい上級者向けには**盤拡大 or 日数短縮**が実効レバー。ただし**収束点コンベンション（実験6）は盤・日数では潰せない**。
5. ★**2人協力版には「中心へ行くだけ」という支配戦略が存在する（実験6, focal 98%）。** これを潰して推理を意味あるものにするには**おじゃま係（第6節）が必須**。おじゃま係は拡張オプションではなく、このゲームを推理ゲームとして成立させる核心。出目開示はこの問題とは無関係なので、消す必要はない（消すと正規の推理プレイが弱くなるだけ）。

## 再現方法

```sh
node sim.js --matrix 10000 --seed=42   # 全実験（約15秒）
node sim.js 10000 2d6 7 7 1 --policy=greedy --share --eps=0.4  # 単発
```
