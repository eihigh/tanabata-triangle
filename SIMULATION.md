# シミュレーション結果（賢いAI編）

引き継ぎ書（DESIGN.md）第4〜5節の計画に基づき、belief（相手位置の確率分布）を持つ賢いAIを実装して実験マトリクスを回した結果。基準構成は **7×7・2d6・7日・減衰1・同マス限定・2人協力**、各条件 10,000 試行（`node sim.js --matrix 10000 --seed=42`）。

## ベースライン再現の確認

まずランダム方策で引き継ぎ書 7.5 節の実測値と照合した:

| 構成 | 今回 | 引き継ぎ書 |
|---|---|---|
| 7×7 出会い率 | 24.6% | 24.9% |
| 5×5 出会い率 | 43.3% | 44.0% |
| 7×7 交差/game | 9.4 | 8.66 |
| 詰み/game | 0.00 | 0.00 |

出会い率はよく一致。交差/game がやや高いのは経路サンプリング方式の差（本実装は「残り歩数で間に合う近傍から無作為」の揺らぐ経路、旧実装はBFS復元）によるもので、ルール自体は同一。**再現OKとみなす。**

## AIの構成

- **belief**: 相手位置の49マス確率分布。初期値は「外周かつ自分から最小距離以上」で一様
- **観測による更新**: ①交差あり→相手経路がそのマスを通った（距離減衰の尤度）、②交差なし→踏んだマスに相手はいない（負の情報・確定除外）、③出会っていない→現在マス除外、④出目開示ありなら相手の移動を正確な遷移行列で前方伝播（なしなら2d6分布で混合）
- **交差は両者に開示**されるため、動いていない側のbeliefも毎手番更新される
- **方策**: greedy（期待距離最小化＋着地マスの的中確率ボーナス）/ infogain（beliefの濃い領域を横断する経路を選ぶ）/ hybrid（1〜3日目infogain→4日目以降greedy）

## 実験1: 方策 × 出目開示（メイン）

| 方策 | 出目なし | 出目あり | 平均決着日 |
|---|---|---|---|
| random | 23.8% | （同一） | 4.03 |
| greedy | **99.1%** | **99.3%** | 2.94 / 2.84 |
| infogain | 97.4% | 95.8% | 3.25 |
| hybrid | 99.1% | 98.6% | 3.22 |

**発見①: 賢さの価値は劇的。** random 24% → greedy 99%。交差＋負の情報（踏んだのに交差しなかったマスの除外）だけで、ベイズ的に推理するペアは7日あればほぼ確実に会える。このゲームは推理がそのまま勝率になる「実力ゲー」であることが確認できた。

**発見②: 出目開示は、最適プレイヤーには ほぼ価値がない。** greedyで+0.2pt、決着が0.1日早まる程度。理由: ベイズ更新をフルに回すAIは出目を知らなくても2d6分布で伝播でき、交差＋負の情報が支配的なため。**ただしこれは「出目開示が無意味」ではない**——人間は49マスの確率分布を頭の中で伝播できない。「交差点＋出目=距離の輪」という人間にも計算できる推理の足場を与えるのが出目開示の本当の役割で、その効果はAIでは測れない（後述のε実験が傍証）。難度をほぼ動かさずに推理を遊びやすくする装置なので、**採用推奨**。

**発見③: infogainは単体では greedy に勝てない。** 交差情報が既に潤沢（98.7%のゲームで交差発生）なので、わざわざ情報を取りに行くより会いに行くほうが強い。hybridもgreedy単体と同等。探索フェーズはこのパラメータでは不要。

## 実験2: 盤サイズ感度（hybrid+出目、7日）

| 盤 | 出会い率 | 平均決着日 |
|---|---|---|
| 5×5 | 99.0% | 2.82 |
| 7×7 | 98.6% | 3.25 |
| 9×9 | 98.0% | 3.64 |

最適プレイでは盤を9×9にしても難度はほぼ動かない（推理が強すぎて吸収される）。

## 実験3: 日数感度（7×7、hybrid+出目）

| 日数 | 出会い率 |
|---|---|
| 5日 | 94.4% |
| 6日 | 96.8% |
| 7日 | 98.6% |
| 8日 | 99.1% |

日数を削っても最適プレイは94%。**目標帯55〜70%を「最適プレイ基準」で狙うのは、日数・盤の調整では不可能**という結論。

## 実験4: 相手モデル v1（ランダム仮定）vs v2（greedy仮定）

| 条件 | v1 | v2 |
|---|---|---|
| greedy | 99.1% | 97.7% |
| greedy+出目 | 99.3% | 98.4% |

v2（相手も自分に寄ってくると仮定して伝播を歪める）はむしろ僅かに悪化。相手の着地は結局サイコロで散るため、ランダム仮定のほうが较正が良い。**v1で十分。**

## 実験5: ε感度＝人間らしい不完全プレイ（7×7、greedy、7日）

確率εで無情報（ランダム）に動くAI。人間の「毎回は完璧に推理できない」をモデル化。

| ε | 出目なし | 出目あり |
|---|---|---|
| 0.0 | 99.1% | 99.3% |
| 0.2 | 94.0% | 93.9% |
| 0.4 | 79.7% | 80.1% |
| 0.6 | **59.4%** | **60.1%** |
| 0.8 | 37.3% | 39.0% |

**目標帯55〜70%は「6割方は勘で動くが、たまに推理を活かす」プレイ（ε≒0.5〜0.6）に相当。** 初見の人間プレイはおそらくε0.5〜0.8帯にいて、上達するほど出会い率が2割→9割超まで伸びる。スキルカーブとして非常に健全。

## 設計への示唆（まとめ）

1. **現行仕様（7×7・2d6・7日・減衰1・同マス）は変更不要。** 人間の実力帯で出会い率25〜95%をフルに使うスキルゲームになっている。目標帯55〜70%は「そこそこ推理できる人」の実測帯として自然に達成される見込み。
2. **出目開示は採用してよい**（プロトタイプに実装済み・トグルあり）。バランスをほぼ動かさず、人間の推理を成立させやすくする。アナログでも口頭で言うだけ。
3. 「簡単になりすぎたら盤拡大・日数短縮で締める」プランは**効かない**（実験2・3）。締めたければ次の一手は**おじゃま係（第6節）**——全知の妨害者は最適ペアの99%を直接崩せる唯一のレバー。
4. 上級者向けに締めたい場合のもう一つの候補は**減衰0.5相当の情報削減**（交差マスの位置を「行のみ」開示等）だが、アナログ運用が崩れるため非推奨。

## 再現方法

```sh
node sim.js --matrix 10000 --seed=42   # 全実験（約25秒）
node sim.js 10000 2d6 7 7 1 --policy=greedy --share --eps=0.6  # 単発
```
